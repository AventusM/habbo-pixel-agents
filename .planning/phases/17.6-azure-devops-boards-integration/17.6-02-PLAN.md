Update `src/extension.ts` to support Azure DevOps as a kanban source:

1. **Add import** at the top of the file:
   ```typescript

   ```

2. **Add Azure DevOps config reader** — create a new function `readAzureDevOpsConfig` (similar structure to `readKanbanConfig`):
   ```typescript
   interface AzureDevOpsConfig {
     organization: string;
     project: string;
     pat: string;
     pollIntervalSeconds: number;
   }

   function readAzureDevOpsConfig(extUri: vscode.Uri): AzureDevOpsConfig {
     const config = vscode.workspace.getConfiguration('habboPixelAgents');
     let organization = config.get<string>('azureDevOps.organization', '');
     let project = config.get<string>('azureDevOps.project', '');
     let pat = config.get<string>('azureDevOps.pat', '');
     const pollIntervalSeconds = config.get<number>('azureDevOps.pollIntervalSeconds', 60);

     if (!organization) {
       try {
         const settingsPath = join(extUri.fsPath, '.vscode', 'settings.json');
         if (existsSync(settingsPath)) {
           const raw = JSON.parse(readFileSync(settingsPath, 'utf8'));
           organization = raw['habboPixelAgents.azureDevOps.organization'] || '';
           project = raw['habboPixelAgents.azureDevOps.project'] || '';
           pat = raw['habboPixelAgents.azureDevOps.pat'] || '';
         }
       } catch {
         // Ignore
       }
     }

     return { organization, project, pat, pollIntervalSeconds };
   }
   ```

3. **Replace the kanban polling section** (lines ~343-354) with a kanbanSource-aware version:

   Read the `kanbanSource` setting: `config.get<string>('kanbanSource', 'github')`.

   If kanbanSource is `'github'`: keep the existing GitHub Projects polling logic (unchanged behavior).

   If kanbanSource is `'azuredevops'`: read Azure DevOps config, and if org + project + pat are all non-empty and pollIntervalSeconds > 0, start an async polling interval:
   ```typescript
   kanbanPollId = setInterval(() => {
     void fetchAzureDevOpsCards(adoOrg, adoProject, adoPat).then((cards) => {
       panel.webview.postMessage({ type: 'kanbanCards', cards } as ExtensionMessage);
     });
   }, adoPollIntervalSeconds * 1000);
   ```

   The `void` prefix + `.then()` pattern avoids unhandled promise rejection (Pitfall 5 from research). The fetchAzureDevOpsCards function already catches all errors internally and returns [].

   Important: the webview already replaces kanbanCards state on each message, so no renderer changes are needed.
