<task type="auto">
  <name>Task 1: Create isoWallRenderer.ts with full wall panel geometry and tests</name>
  <files>src/isoWallRenderer.ts, tests/isoWallRenderer.test.ts</files>
  <action>
Create `src/isoWallRenderer.ts` with a single exported function `drawWallPanels` that draws proper room-perimeter walls:

1. **Scan the grid for wall edges:**
   - Left wall tiles: tiles where `tx === 0` OR left neighbor (`tx-1`) is void, for each row `ty`
   - Right wall tiles: tiles where `ty === 0` OR top neighbor (`ty-1`) is void, for each col `tx`

2. **Compute shared baseline:**
   - For left wall edges, find the maximum `screenY + TILE_H` across all left-edge tiles in the grid (accounting for tile height). This is the lowest point. All left wall strips extend down to `maxScreenY + WALL_HEIGHT` so there are no gaps.
   - Same for right wall edges.

3. **Draw wall strips per edge tile:**
   - For each left-edge tile at `(tx, ty)` with height `tileZ`: compute screen position via `tileToScreen(tx, ty, tileZ)`. Draw a parallelogram from the tile's left edge down to the shared baseline using `tileColors(hsb).left` fill color.
   - Left wall parallelogram vertices:
     - Top-left: `(sx - TILE_W_HALF, sy + TILE_H_HALF)`
     - Top-right: `(sx, sy + TILE_H)`
     - Bottom-right: `(sx, baseline)`
     - Bottom-left: `(sx - TILE_W_HALF, baseline - TILE_H_HALF)` — NO, this must maintain the parallelogram shape. Actually, the wall strip extends straight DOWN, so:
     - Top-left: `(sx - TILE_W_HALF, sy + TILE_H_HALF)`
     - Top-right: `(sx, sy + TILE_H)`
     - Bottom-right: `(sx, sy + TILE_H + stripHeight)` where `stripHeight = baseline - (sy + TILE_H)`
     - Bottom-left: `(sx - TILE_W_HALF, sy + TILE_H_HALF + stripHeight)`
   - Same logic mirrored for right wall strips using `tileColors(hsb).right`.

4. **Back corner post:**
   - At tile `(0, 0)`: draw a thin vertical rectangle (4px wide) from `tileToScreen(0, 0, tileZ).y` down `WALL_HEIGHT` pixels in the `left` color. Only draw if tile `(0,0)` is non-void.

5. **Function signature:**
```typescript
export function drawWallPanels(
  ctx: CanvasRenderingContext2D | OffscreenCanvasRenderingContext2D,
  grid: TileGrid,
  cameraOrigin: { x: number; y: number },
  hsb: HsbColor,
  tileColorMap?: Map<string, HsbColor>,
): void;
```

6. **Wall drawing order:** Draw walls BEFORE floor tiles in the depth sort (walls are behind everything). Since walls are on the OffscreenCanvas, draw them first, then draw the depth-sorted floor tiles on top.

For the test file `tests/isoWallRenderer.test.ts`:
- Test that `drawWallPanels` is callable with a simple 3x3 grid (no throw)
- Test that a grid with void tiles correctly identifies wall edges
- Test baseline computation: for a grid with tiles at different heights, verify the wall extends to the correct bottom position
- Use the existing setup.ts mocks (OffscreenCanvas mock already exists)
