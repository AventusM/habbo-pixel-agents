  <name>Task 1: Create KanbanCard type, githubProjects.ts fetch module, and message protocol extension</name>
  <files>src/agentTypes.ts, src/githubProjects.ts, tests/githubProjects.test.ts</files>
  <action>
1. **Update `src/agentTypes.ts`:**
   - Add `KanbanCard` interface:
     ```typescript
     export interface KanbanCard {
       id: string;
       title: string;
       status: string; // raw status column name from GitHub Projects
     }
     ```
   - Extend `ExtensionMessage` union with:
     ```typescript
     | { type: 'kanbanCards'; cards: KanbanCard[] }
     ```

2. **Create `src/githubProjects.ts`:**
   - Import `execFileSync` from `child_process`, `writeFileSync` from `fs`, `tmpdir` from `os`, `join` from `path`.
   - Import `KanbanCard` from `./agentTypes.js`.
   - Export `fetchKanbanCards(owner: string, projectNumber: number, ownerType: 'org' | 'user' = 'org'): KanbanCard[]`:
     - Step 1: Build GraphQL query to get project node ID. Use `organization(login: "...")` for org, `user(login: "...")` for user owner type.
     - Step 2: Execute `execFileSync('gh', ['api', 'graphql', '-f', 'query=...'])` to get project ID. Use `execFileSync` with args array (NOT `execSync` with shell string) to avoid shell quoting issues.
     - Step 3: Build items query (fetch first 100 items with content title and fieldValues for Status field). Write query to temp file via `writeFileSync` to avoid shell quoting with complex GraphQL. Use `execFileSync('gh', ['api', 'graphql', '--input', tmpFile])`.
     - Step 4: Parse response, extract title from `content` (Issue, PullRequest, or DraftIssue), extract status from `fieldValues.nodes` where `field.name === 'Status'`.
     - Step 5: Return `KanbanCard[]`.
     - **Error handling:** Wrap entire function in try/catch. On ANY error (gh not installed, auth failure, network error, parse error), log to console and return `[]`. Follow the audio silent-fallback pattern from Phase 8.

3. **Create `tests/githubProjects.test.ts`:**
   - Mock `child_process.execFileSync` using `vi.mock`.
   - Test: when `execFileSync` throws (gh not installed), `fetchKanbanCards` returns `[]`.
   - Test: when `execFileSync` returns valid JSON response with 2 items (one Issue, one DraftIssue), cards are correctly parsed with titles and statuses.
   - Test: when an item has no Status field value, status defaults to `'No Status'`.
   - Test: when content title is null (deleted issue), title defaults to `'(no title)'`.
  </action>
  <verify>npx vitest run tests/githubProjects.test.ts</verify>
  <done>KanbanCard type exported from agentTypes; fetchKanbanCards handles success and all error cases; 4+ tests pass</done>
