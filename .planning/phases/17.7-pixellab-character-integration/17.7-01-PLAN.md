# Phase 17.7: PixelLab Character Integration

## Goal
Replace the multi-layer Nitro Habbo avatar rendering with single-sprite PixelLab-generated characters.

## Context
- Current avatar system uses 13-layer body part composition (body, head, clothing, hair, etc.)
- PixelLab MCP server generates pre-rendered pixel art characters with 8 directional views and animations
- Switching to PixelLab simplifies the rendering pipeline significantly

## Tasks

### Task 1: Create sprite packer script
- **File**: `scripts/pack-pixellab-sprites.mjs`
- **What**: Node.js script that packs individual PixelLab character PNGs into a single TexturePacker-compatible spritesheet + JSON manifest
- **Details**:
  - Reads extracted PixelLab ZIP directory
  - Maps PixelLab directions (south, south-west, etc.) to Habbo directions (0-7)
  - Mirrors west-side sprites for missing east-side directions
  - Falls back to static rotation for completely missing animation directions
  - Outputs `assets/pixellab/{name}.png` and `assets/pixellab/{name}.json`

### Task 2: Generate character sprites
- **Character**: "Beanie Hoodie Guy" — black beanie, yellow hoodie, dark pants, sneakers
- **Animations**: breathing-idle (4 frames), walking (6 frames), running (6 frames)
- **Output**: `assets/pixellab/beanie-hoodie-guy.png` (576x576, 136 frames)

### Task 3: Update build pipeline
- **File**: `esbuild.config.mjs`
- **What**: Copy `assets/pixellab/` to `dist/webview-assets/pixellab/` during build

### Task 4: Wire asset loading
- **File**: `src/extension.ts` — Add pixellab PNG/JSON URIs to ASSET_URIS
- **File**: `src/webview.tsx` — Load pixellab atlas via `spriteCache.loadAtlas()`

### Task 5: Create PixelLab avatar renderer
- **File**: `src/isoAvatarRenderer.ts`
- **What**: Add `createPixelLabAvatarRenderable()` function
- **Details**:
  - Single sprite draw per frame (no multi-layer composition)
  - Frame key format: `pl_{animation}_{habboDirection}_{frameNum}`
  - 6-frame walk cycle (vs 4-frame Nitro)
  - 4-frame breathing idle animation
  - Spawn/despawn clip effect preserved
  - `usePixelLab` flag on AvatarSpec

### Task 6: Switch rendering pipeline
- **File**: `src/RoomCanvas.tsx`
- **What**: Prioritize PixelLab renderer, fall back to Nitro/placeholder

### Task 7: Fix pre-existing type error
- **File**: `src/agentTypes.ts`
- **What**: Add missing `exportDebugGrid` to `WebviewMessage` union type

## Success Criteria
- All avatars render as PixelLab Beanie Hoodie Guy character
- Walking animation plays smoothly with 6-frame cycle
- Idle breathing animation cycles in all 8 directions
- Spawn/despawn effects work correctly
- Build completes without errors
- Falls back gracefully if PixelLab atlas fails to load
