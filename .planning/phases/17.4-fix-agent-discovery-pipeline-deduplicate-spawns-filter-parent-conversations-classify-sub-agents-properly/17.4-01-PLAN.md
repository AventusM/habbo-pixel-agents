**Bug 2 fix — Filter parent conversations in discoverAgents():**

In `discoverAgents()`, remove the loop that tracks root-level JSONL files (lines 57-69 approximately — the first `for (const file of files)` loop that checks `file.endsWith('.jsonl')`). Root-level JSONL files are always parent/orchestrator conversations and should never spawn avatars. Only files inside `subagents/` directories are actual working agents.

Also update the `fs.watch` callback for the claude directory (lines 110-134): when a new `.jsonl` file appears in the root directory, do NOT call `trackAgent`. Only track files discovered via subagents directories.

Specifically:
1. Delete or comment out the first `for (const file of files)` loop that tracks root `.jsonl` files
2. In the `fs.watch` callback, remove the `if (filename.endsWith('.jsonl'))` branch that calls `this.trackAgent` for root-level files

**Bug 3 fix — Read meta.json for classification in trackAgent():**

In `trackAgent(jsonlPath)`, before calling `extractSubagentType(initialContent)`, check for a companion `.meta.json` file:

```typescript
// Try meta.json first (authoritative source from Claude Code)
const metaPath = jsonlPath.replace('.jsonl', '.meta.json');
let subagentType: string | null = null;
try {
  if (fs.existsSync(metaPath)) {
    const meta = JSON.parse(fs.readFileSync(metaPath, 'utf8'));
    subagentType = meta.agentType || null;
  }
} catch {}

// Fall back to JSONL content scanning only if meta.json unavailable
if (!subagentType) {
  subagentType = extractSubagentType(initialContent);
}
```

Replace the current line `const subagentType = extractSubagentType(initialContent);` with the above pattern.

**Bug 1 fix — Deduplication guard:**

In `src/RoomCanvas.tsx`, find the `case 'agentCreated'` handler (around line 265). Add an early-return guard at the very top of the case block, before any spawn or side-effect logic:

```typescript
case 'agentCreated': {
  // Guard: skip if avatar already exists (prevents duplicate side effects from re-broadcast)
  if (avatarManager.getAvatar(msg.agentId)) {
    console.log(`[Room] agentCreated: ${msg.agentId} already exists, skipping`);
    break;
  }
  // ... existing spawn logic unchanged
}
```

This prevents `orchestrationAddAgent`, `sectionManager.assignAgent`, `idleWander.setAgentRole`, and `idleWander.startWandering` from running again on duplicate `agentCreated` messages (which happen when the webview sends `requestAgents` and the extension re-broadcasts all agents).

Important: The `avatarManager.spawnAvatarAt` already returns existing avatars without re-creating them, but ALL the side effects after it still run. The guard must be BEFORE the spawn call to skip everything.

Note: The `reassignAgent` method in agentManager.ts intentionally re-sends `agentCreated` for team changes — this is fine because `reassignAgent` is only called for explicit team reassignment, and the avatar already exists so the guard will skip it. If team reassignment needs to update the avatar's team in the future, that should be a separate message type. For now, reassignment works via the webview receiving the message and updating its own state (which already happens before this handler).
