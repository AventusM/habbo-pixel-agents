```typescript
export interface MultiTileFurnitureSpec {
  name: string;
  tileX: number;         // Origin tile (bottom-left of footprint)
  tileY: number;
  tileZ: number;
  widthTiles: number;    // Footprint width (e.g., 2 for 2×1 desk)
  heightTiles: number;   // Footprint height (e.g., 1 for 2×1 desk)
  direction: 0 | 2 | 4 | 6;
}
```

2. **createMultiTileFurnitureRenderable() function** — core logic:
   - **CRITICAL: Calculate max coordinate across full footprint:**
     ```typescript
     const sortTileX = spec.tileX + spec.widthTiles - 1;
     const sortTileY = spec.tileY + spec.heightTiles - 1;
     ```
   - Return Renderable with `tileX: sortTileX, tileY: sortTileY` (for depth sort)
   - **Render at ORIGIN tile, not sort tile:**
     ```typescript
     const screen = tileToScreen(spec.tileX, spec.tileY, spec.tileZ);
     // NOT tileToScreen(sortTileX, sortTileY, ...)
     ```
   - Frame lookup, coordinate rounding, direction mirroring — same as single-tile pattern
   - Graceful null handling if sprite missing

3. **Why max coordinate?** (per 04-RESEARCH.md Pitfall 2):
   - Origin tile (minX, minY) causes avatar occlusion bug — avatars behind desk appear in front
   - Max coordinate (maxX, maxY) ensures desk's farthest edge determines depth
   - Depth sort formula `tileX + tileY + tileZ * 0.001` — higher X+Y renders later (on top)
   - Desk at (3,3) with 2×1 footprint: origin is (3,3), max is (4,3) → uses (4,3) for sort key

   - Test 10: Multi-tile with direction 4 applies horizontal flip correctly
