Create src/isoFurnitureRenderer.ts implementing single-tile furniture rendering using the pattern from 04-RESEARCH.md "Pattern 1: Single-Tile Furniture Rendering."

Implementation requirements:

1. **FurnitureSpec interface** — define:
```typescript
export interface FurnitureSpec {
  name: string;        // e.g., "chair"
  tileX: number;
  tileY: number;
  tileZ: number;
  direction: 0 | 2 | 4 | 6;  // Habbo 4-direction system
}
```

2. **createFurnitureRenderable() function** — returns Renderable object:
   - Frame key format: `${spec.name}_64_a_${baseDirection}_0`
   - Base direction mapping: direction 4 → 2, direction 6 → 0 (mirroring pattern)
   - Call `spriteCache.getFrame(atlasName, frameKey)` to get SpriteFrame
   - Graceful null check: if frame missing, log warning and return early
   - Convert tile position to screen using `tileToScreen(spec.tileX, spec.tileY, spec.tileZ)`
   - **CRITICAL: Round coordinates** with `Math.floor()` before drawImage to avoid sub-pixel rendering cost
   - Center sprite horizontally: `dx = Math.floor(screen.x - frame.w / 2)`
   - Align bottom to tile: `dy = Math.floor(screen.y - frame.h)`
   - For directions 4 and 6: apply horizontal flip using `ctx.scale(-1, 1)` and adjust dx to `-dx - frame.w`

3. **Direction mapping helpers** (optional):
```typescript
function getBaseDirection(direction: 0 | 2 | 4 | 6): 0 | 2;
function shouldMirrorSprite(direction: 0 | 2 | 4 | 6): boolean;
```

4. **Assume zero anchor offset** — start with `offsetX = 0, offsetY = 0`. Defer anchor offset logic to visual validation step. Most 64×64 furniture sprites are untrimmed.

5. **Unit tests** in tests/isoFurnitureRenderer.test.ts:
   - Test 1: createFurnitureRenderable returns Renderable with correct tileX/Y/Z
   - Test 2: Frame key format matches pattern for direction 0
   - Test 3: Direction 4 uses base direction 2 (mirroring)
   - Test 4: Direction 6 uses base direction 0 (mirroring)
   - Test 5: Missing sprite frame returns renderable with no-op draw (no crash)

Use existing Phase 2/3 test patterns — mock SpriteCache, mock OffscreenCanvas context.

**DO NOT** implement multi-tile furniture in this task — that is Plan 04-02. Single-tile only (chair, lamp, etc.).

**DO NOT** import React or DOM modules — this is a pure rendering module like isoTypes.ts.

Reference 04-RESEARCH.md "Pattern 1: Single-Tile Furniture Rendering" for complete code example.
