<task type="auto">
  <name>Task 1: Create camera controller module</name>
  <files>src/cameraController.ts, tests/cameraController.test.ts</files>
  <action>
    1. Create `src/cameraController.ts`:
       - `CameraState` interface: `{ panX: number; panY: number; zoom: number; isDragging: boolean; dragStartX: number; dragStartY: number; dragStartPanX: number; dragStartPanY: number }`
       - `createCameraState(): CameraState` — returns default state with panX=0, panY=0, zoom=1.0
       - `applyPan(state: CameraState, dx: number, dy: number): void` — mutates panX/panY (mutable for renderState ref pattern)
       - `applyZoom(state: CameraState, delta: number, pivotX: number, pivotY: number, canvasWidth: number, canvasHeight: number): void` — adjusts zoom with pivot point correction so zoom centers on mouse position. Clamp zoom between 0.3 and 2.0.
       - `applyCameraTransform(ctx: CanvasRenderingContext2D, state: CameraState, canvasWidth: number, canvasHeight: number): void` — applies `ctx.translate(canvasWidth/2, canvasHeight/2); ctx.scale(zoom, zoom); ctx.translate(-canvasWidth/2 + panX, -canvasHeight/2 + panY)`.
       - `screenToWorld(screenX: number, screenY: number, state: CameraState, canvasWidth: number, canvasHeight: number): { x: number; y: number }` — inverse transform for mouse-to-tile conversion, accounting for pan and zoom.
       - `jumpToSection(state: CameraState, sectionCenterX: number, sectionCenterY: number, canvasWidth: number, canvasHeight: number): void` — sets panX/panY to center the given world position.

    2. Create `tests/cameraController.test.ts`:
       - Default state has zoom=1.0, panX=0, panY=0
       - applyPan(state, 10, 20) updates panX/panY correctly
       - applyZoom clamps between 0.3 and 2.0
       - screenToWorld at zoom=1.0 and no pan returns correct world coords
       - screenToWorld at zoom=2.0 accounts for scaling
       - jumpToSection sets panX/panY to center the target position
  </action>
  <verify>npx vitest run tests/cameraController.test.ts -x</verify>
  <done>Camera controller module passes all tests with correct pan/zoom math</done>

  <action>
    Modify `src/RoomCanvas.tsx` to use the camera controller:

    1. Add `cameraState` to the `renderState` ref (type CameraState, initialized via createCameraState()).

    2. Add mouse event handlers for camera:
       - `onMouseDown`: If NOT in editor mode (view mode), set isDragging=true, record dragStartX/Y and dragStartPanX/Y. Use middle-click OR left-click with no avatar hit as drag trigger. To avoid conflicting with avatar clicks, use a `dragThreshold` of 5px — only start pan after mouse moves >5px from mousedown position.
       - Update `handleMouseMove`: If isDragging, compute dx/dy from dragStart, call applyPan.
       - `onMouseUp`: Set isDragging=false.
       - `onWheel`: Call applyZoom with event.deltaY, pivot at mouse position.

    3. Update `frame()` render loop:
       - After `ctx.clearRect`, call `ctx.save(); applyCameraTransform(ctx, cameraState, ...); ctx.drawImage(offscreen, 0, 0);` then all dynamic renderables drawn inside the camera transform. `ctx.restore()` after all world-space drawing.
       - Remove the per-avatar `ctx.translate(cameraOrigin.x, cameraOrigin.y)` calls — camera transform handles this now.
       - UI overlays (speech bubbles, name tags) also drawn inside camera transform since they're world-positioned.

    4. Update `getHoveredTile` calls: Pass mouse coordinates through `screenToWorld()` before passing to the tile conversion function. This means adjusting the mouse position in handleMouseMove, handleClick, handleContextMenu to account for camera transform.

    5. Keep existing `cameraOrigin` for the initial centering offset — it computes where tile (0,0) starts. The new camera panX/panY is an additional offset on top of that.
