  <files>src/sectionManager.ts, src/webview.tsx</files>
  <action>
    1. Create `src/sectionManager.ts`:
       - `SectionState` interface: `{ team: TeamSection; agentIds: string[]; activityLevel: number; lastActivityMs: number }`
       - `SectionManager` class:
         - `constructor(template: FloorTemplate)` — initializes 4 SectionState entries from template
         - `assignAgent(agentId: string, team: TeamSection): void` — adds agent to section
         - `removeAgent(agentId: string): void` — removes from all sections
         - `getAgentTeam(agentId: string): TeamSection | null` — returns team the agent belongs to (needed for despawn to find the correct booth)
         - `getSpawnTile(team: TeamSection): {x,y} | null` — returns teleportTile from section layout
         - `getDeskTile(team: TeamSection, existingOccupied: Set<string>): {x,y,dir} | null` — returns first unoccupied desk tile
         - `getIdleTile(team: TeamSection): {x,y} | null` — returns random idle tile within section
         - `getSectionAgentCount(team: TeamSection): number`
         - `getTemplate(): FloorTemplate`

    2. Update `src/webview.tsx`:

       - Make template globally accessible: `(window as any).floorTemplate = template;`
       - Pass `FLOOR_HEIGHTMAP` to `RoomCanvas` instead of `DEMO_HEIGHTMAP`
       - Add VS Code settings for template size: listen for `{ type: 'templateSize'; size: string }` message from extension, re-render with new template

<task type="auto">
  <name>Task 2: Section-aware spawning, despawning, and teleport effects in RoomCanvas</name>
  <files>src/RoomCanvas.tsx, src/avatarManager.ts</files>
  <action>
    1. Update `src/avatarManager.ts`:
       - Add `spawnAvatarAt(agentId, variant, tileX, tileY, tileZ, grid, displayName): AvatarSpec | null` — spawns avatar at a specific tile (the teleport booth position) instead of random walkable tile. Reuse existing spawn logic but override tile selection.
       - Keep `spawnAvatar` as fallback for backward compatibility.

    2. Update `src/RoomCanvas.tsx`:
       - Import `SectionManager`, `createTeleportEffect`, `drawTeleportFlash`, `isEffectActive`
       - Add `sectionManagerRef` to component refs, initialized from `(window as any).floorTemplate`
       - Add `teleportEffectsRef = useRef<TeleportEffect[]>([])` for active effects
       - Add `despawningAgentsRef = useRef<Set<string>>(new Set())` to track agents mid-despawn

    3. Update `agentCreated` handler in the extensionMessage listener:
       - Extract `team` from the message (added in Plan 05)
       - Look up spawn tile via `sectionManager.getSpawnTile(team)`
       - Call `avatarManager.spawnAvatarAt(agentId, variant, spawnTile.x, spawnTile.y, 0, grid, displayName)`
       - Create teleport effect: `teleportEffectsRef.current.push(createTeleportEffect(screenX, screenY, 'spawn'))`
       - Assign agent to section: `sectionManager.assignAgent(agentId, team)`

    4. **Despawn flow — handle `agentRemoved` message (CONTEXT.md locked decision: walk-to-booth despawn):**
       - On `agentRemoved` message:
         a. Look up the agent's team via `sectionManager.getAgentTeam(agentId)`
         b. Get the teleport booth tile via `sectionManager.getSpawnTile(team)`
         c. Add agentId to `despawningAgentsRef.current` (prevents idle wander from overriding the walk target)
         d. Pathfind the avatar to the teleport booth tile using the existing pathfinding/walk system
         e. When the avatar reaches the booth tile (check in `frame()` or via walk-complete callback):
            - Create despawn teleport effect: `teleportEffectsRef.current.push(createTeleportEffect(screenX, screenY, 'despawn'))`
            - After effect completes (~500ms), remove the avatar: `avatarManager.removeAvatar(agentId)`
            - Remove from section: `sectionManager.removeAgent(agentId)`
            - Remove from despawning set: `despawningAgentsRef.current.delete(agentId)`
       - Edge case: if agent has no team (fallback), remove immediately without walk animation

    5. Update `agentStatus` handler:
       - Skip status updates for agents in `despawningAgentsRef.current` (they are walking to booth, not responding to status changes)
       - When status='active': look up desk tile via `sectionManager.getDeskTile(team, occupiedDesks)` instead of hardcoded `DESK_TILES`
       - When status='idle': wander within section bounds using `sectionManager.getIdleTile(team)` as wander target

    6. Update `frame()` render loop:
       - After drawing avatars and before UI overlays, draw active teleport effects:
         ```typescript
         teleportEffectsRef.current = teleportEffectsRef.current.filter(
           effect => drawTeleportFlash(ctx, effect, currentTimeMs)
         );
         ```
       - Check despawning agents: for each agent in despawningAgentsRef, if avatar has reached booth tile, trigger despawn effect and schedule removal

    7. Remove hardcoded `DESK_TILES` array — replaced by section-aware desk lookup.
  </action>
