<task type="auto">
  <name>Task 1: Create message bridge and sidebar panel provider</name>
  <files>src/messageBridge.ts, src/orchestrationPanel.ts, tests/messageBridge.test.ts, package.json</files>
  <action>
    1. Create `src/messageBridge.ts`:
       - `MessageBridge` class:
         - `private roomPanel: vscode.WebviewPanel | null`
         - `private sidePanel: vscode.WebviewView | null`
         - `setRoomPanel(panel: vscode.WebviewPanel): void`
         - `setSidePanel(view: vscode.WebviewView): void`
         - `sendToRoom(msg: any): void` — posts message to room webview
         - `sendToSidebar(msg: any): void` — posts message to sidebar webview
         - `broadcastAgentEvent(msg: ExtensionMessage): void` — sends to BOTH room and sidebar
         - `handleRoomMessage(msg: any): void` — processes messages from room, relays to sidebar if needed (e.g., agentClicked)
         - `handleSidebarMessage(msg: any): void` — processes messages from sidebar, relays to room if needed (e.g., jumpToSection with target coordinates)

    2. Create `src/orchestrationPanel.ts`:
       - `OrchestrationPanelProvider` implements `vscode.WebviewViewProvider`:
         - `constructor(extensionUri: vscode.Uri, messageBridge: MessageBridge)`
         - `resolveWebviewView(webviewView)`: sets enableScripts, localResourceRoots, HTML content via getOrchestrationPanelHtml, registers message handler
         - Message handler forwards sidebar messages to `messageBridge.handleSidebarMessage()`
       - Export the class for registration in extension.ts

    3. Update `package.json` contributes section:
       - Add `viewsContainers.activitybar`: `[{ "id": "habbo-orchestration", "title": "Habbo Agents", "icon": "media/habbo-icon.svg" }]`
       - Add `views.habbo-orchestration`: `[{ "type": "webview", "id": "habboPixelAgents.orchestrationPanel", "name": "Orchestration" }]`
       - Create a simple placeholder SVG icon: write a minimal 24x24 SVG with a pixel art style icon to `media/habbo-icon.svg`

    4. Create `tests/messageBridge.test.ts`:
       - broadcastAgentEvent sends to both panels (mock webviews)
       - handleSidebarMessage('jumpToSection') relays to room
       - handleRoomMessage('agentClicked') relays to sidebar
       - Null panels don't throw (graceful when panel not yet created)
  </action>

  <done>MessageBridge relays between room and sidebar, OrchestrationPanelProvider registered, package.json updated with viewsContainers</done>
