**Modify extension.ts activate() function:**

After creating the WebviewPanel and before setting html property:

1. **Generate asset URIs for chair atlas:**

```typescript
// After: const panel = vscode.window.createWebviewPanel(...)

// Generate webview URIs for chair atlas (proof-of-concept)
const chairPngUri = panel.webview.asWebviewUri(
  vscode.Uri.joinPath(context.extensionUri, 'dist', 'webview-assets', 'chair_atlas.png')
);
const chairJsonUri = panel.webview.asWebviewUri(
  vscode.Uri.joinPath(context.extensionUri, 'dist', 'webview-assets', 'chair_atlas.json')
);

console.log('Chair PNG URI:', chairPngUri.toString());
console.log('Chair JSON URI:', chairJsonUri.toString());
```

2. **Update CSP meta tag to include img-src:**

Change existing CSP from:
```typescript
content="default-src 'none'; script-src ${panel.webview.cspSource}; style-src 'unsafe-inline';"
```

To:
```typescript
content="default-src 'none'; script-src ${panel.webview.cspSource}; img-src ${panel.webview.cspSource}; style-src 'unsafe-inline';"
```

This allows the webview to load images from `vscode-resource://` URIs (per Pitfall 3 from research).

3. **Pass asset URIs to webview via inline script:**

Add before the closing `</head>` tag in HTML template:

```typescript
<script>
  window.ASSET_URIS = {
    chairPng: '${chairPngUri}',
    chairJson: '${chairJsonUri}'
  };
</script>
```

This makes asset URIs available in webview.tsx without requiring postMessage.

**Rationale:**
- asWebviewUri() converts file:// paths to vscode-resource:// URIs that pass security checks
- img-src CSP directive allows image loads (without it, all image loads fail with CSP violation)
- Inline ASSET_URIS avoids async postMessage complexity for initial asset load
- Console.log outputs help verify URIs are generated correctly (debugging aid)

**Add chair atlas loading after React root render:**

```typescript
// After: createRoot(document.getElementById('root')!).render(...)

// Initialize sprite cache and load chair atlas
const spriteCache = new SpriteCache();

(async () => {
  try {
    const { chairPng, chairJson } = (window as any).ASSET_URIS;

    console.log('Loading chair atlas from:', chairPng, chairJson);

    await spriteCache.loadAtlas('chair', chairPng, chairJson);

    console.log('✓ Chair atlas loaded successfully');

    // Test frame lookup
    const frame = spriteCache.getFrame('chair', 'chair_64_a_0_0');
    if (frame) {
      console.log('✓ Frame lookup succeeded:', {
        x: frame.x,
        y: frame.y,
        w: frame.w,
        h: frame.h,
        bitmapWidth: frame.bitmap.width,
        bitmapHeight: frame.bitmap.height,
      });
    } else {
      console.warn('Frame lookup returned null — check frame name in manifest');
    }
  } catch (error) {
    console.error('Asset loading failed:', error);
  }
})();
```

**Rationale:**
- IIFE async function allows await without top-level await (broader compatibility)
- Console logs provide verification that asset pipeline works end-to-end
- Frame lookup test confirms sprite cache integration
- Error handling prevents crashes if assets fail to load
