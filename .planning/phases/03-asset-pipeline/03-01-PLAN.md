Create TypeScript module with three exports:

**1. SpriteManifest interface** (Texture Packer JSON Hash format):
```typescript
export interface SpriteManifest {
  frames: Record<string, {
    frame: { x: number; y: number; w: number; h: number };
    rotated: boolean;
    trimmed: boolean;
    spriteSourceSize: { x: number; y: number; w: number; h: number };
    sourceSize: { w: number; h: number };
  }>;
  meta: {
    image: string;
    format: string;
    size: { w: number; h: number };
  };
}
```

**2. SpriteFrame interface** (frame lookup return type):
```typescript
export interface SpriteFrame {
  bitmap: ImageBitmap;
  x: number;
  y: number;
  w: number;
  h: number;
}
```

**3. SpriteCache class** with methods:

- `async loadAtlas(atlasName: string, pngUri: string, jsonUri: string): Promise<void>`
  - Load PNG as HTMLImageElement via `new Image()` with src=pngUri
  - Wait for load via Promise wrapper around img.onload/onerror
  - Immediately call `createImageBitmap(img)` after load (GPU pre-decode)
  - Store ImageBitmap in `private bitmaps: Map<string, ImageBitmap>`
  - Fetch JSON manifest from jsonUri via `fetch()`, parse as SpriteManifest
  - Store manifest in `private manifests: Map<string, SpriteManifest>`
  - Return void (no value returned, only stores in cache)

- `getFrame(atlasName: string, frameName: string): SpriteFrame | null`
  - Retrieve bitmap and manifest from Maps
  - If either missing, return null (graceful degradation)
  - Look up frameData from `manifest.frames[frameName]`
  - If frame not in manifest, return null
  - Return SpriteFrame object with bitmap and frame.{x,y,w,h}

- `dispose(): void`
  - Iterate all bitmaps in cache, call `bitmap.close()` to free GPU memory
  - Clear both Maps

Use Map data structures (not plain objects) for O(1) lookup performance.

**Error handling:** If image load fails or JSON fetch fails, let Promise reject naturally — caller handles. If frame not found in manifest, return null (not error).

**Type safety:** All parameters and returns fully typed, no `any` types.

Follow existing codebase patterns:
- ESM imports with .js extensions
- Strict TypeScript (no loose nulls)
- Zero DOM imports beyond HTMLImageElement/ImageBitmap (browser built-ins)
