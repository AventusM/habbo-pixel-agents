```typescript
function getTintCanvas(w: number, h: number): OffscreenCanvasRenderingContext2D {
  if (!_tintCanvas || _tintCanvas.width < w || _tintCanvas.height < h) {
```

**Root cause (HIGH confidence):** The tint canvas is shared (min 128x128) but `clearRect` only clears `(0, 0, frame.w, frame.h)`. When a small face sprite (e.g., 4x5 eyes at dir 1) is drawn after a large body sprite (e.g., 28x28 head), residual tinted pixels from the previous sprite remain outside the 4x5 clear area. The `destination-in` compositing SHOULD mask these, but the multiply fill at step 2 covers the full `frame.w x frame.h` area — not the full canvas. So residual pixels outside the multiply rect but inside the canvas survive the destination-in step (they have alpha from a prior draw and were not overwritten). These residual pixels then get blitted to the main canvas, appearing as a stray particle offset from the avatar.
