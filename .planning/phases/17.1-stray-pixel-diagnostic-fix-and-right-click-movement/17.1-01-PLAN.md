<task type="auto">
  <name>Task 1: Create spritesheet stray pixel scanner and fix face PNG</name>
  <files>scripts/check-spritesheet-stray-pixels.mjs</files>
  <action>
Create a Node.js script that scans figure spritesheet PNGs for stray alpha pixels outside declared frame bounds.

1. **Create `scripts/check-spritesheet-stray-pixels.mjs`** that:
   - Reads each figure JSON in `assets/habbo/figures/`
   - Loads the corresponding PNG using `sharp` or raw PNG decoding (use `pngjs` — already available or install)
   - For each pixel in the PNG, checks if it falls within ANY declared spritesheet frame's bounding box
   - Reports any pixel with alpha > 0 that is NOT within any frame's bounds
   - Also reports any frame that contains unexpected alpha pixels at its borders (possible bleed from neighbors)

2. **Run the scanner** on `hh_human_face.png` specifically:
   ```bash
   node scripts/check-spritesheet-stray-pixels.mjs assets/habbo/figures/hh_human_face
   ```

3. **If stray pixels found in the PNG:** The convert script (`scripts/convert-cortex-to-nitro.mjs`) may need a padding pass, OR manually clear the stray pixels by adding a post-processing step to the conversion. The simplest fix: add 1px transparent padding between frames in the spritesheet packer, OR scan and zero-out any alpha pixels outside frame bounds after packing.

4. **If NO stray pixels in PNG:** The issue is in canvas compositing. In that case, change the tint approach in `drawTintedBodyPart` to avoid the multiply alpha bleed:
   - After Step 3 (destination-in), add an explicit clip to the sprite bounds before blitting:
   ```typescript
   // Step 3.5: Explicitly zero alpha outside sprite bounds
   tCtx.globalCompositeOperation = "destination-in";
   tCtx.fillStyle = "#000"; // Color doesn't matter, only alpha
   tCtx.fillRect(0, 0, frame.w, frame.h);
   ```
   This ensures any residual alpha outside the multiply rect is cleared.
  </action>
