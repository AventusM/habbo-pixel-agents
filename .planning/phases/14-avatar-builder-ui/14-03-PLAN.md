**1. Create `src/avatarBuilderPreview.ts`:**

A standalone preview renderer that draws a single avatar at a fixed position on a small canvas, facing direction 2 (front-facing, classic Habbo builder style).

```typescript
import type { OutfitConfig } from './avatarOutfitConfig.js';
import type { SpriteCache, NitroSpriteFrame } from './isoSpriteCache.js';
import { outfitToFigureParts } from './avatarOutfitConfig.js';
import { TILE_H_HALF, TILE_W_HALF } from './isometricMath.js';

export function renderAvatarPreview(
  ctx: CanvasRenderingContext2D,
  outfit: OutfitConfig,
  spriteCache: SpriteCache,
  variant: number,
): void { ... }
```

Implementation:
- Clear the canvas
- Set `ctx.imageSmoothingEnabled = false`
- Use direction 2 (front-facing), frame 0 (idle)
- Build the same 11-layer part mapping as the main renderer: call `outfitToFigureParts(outfit)` to get figureParts
- For each part in render order (use the RENDER_ORDER_LEFT_BEHIND since dir 2 means left arm is far side): build the frame key using the same `h_std_{part}_{setId}_2_0` format
- Get the frame from spriteCache.getNitroFrame
- Replicate the tinting logic: draw sprite to a small offscreen canvas, multiply with color, destination-in to restore alpha, draw result to the preview canvas
- Center the avatar in the preview canvas (e.g., 120x180 canvas, avatar centered at x=60, y=140)
- Head shape: setId = (variant % 4) + 1, same as main renderer

The tinting logic is the same as `drawTintedBodyPart` in isoAvatarRenderer.ts. Since that function is not exported, re-implement it locally in this file (a lightweight version — no spawn clipping, no walk animation). This is acceptable duplication because the preview is intentionally simpler and decoupled from the room render loop (per research anti-pattern guidance).

Export `renderAvatarPreview` and the preview canvas dimensions as constants:
```typescript
export const PREVIEW_WIDTH = 120;
export const PREVIEW_HEIGHT = 180;
```

**2. Create `src/AvatarBuilderModal.tsx`:**

A React modal component rendered as an overlay `<div>` with `position: fixed` and high z-index, displayed above the room canvas.

Props interface:
```typescript
interface AvatarBuilderModalProps {
  avatarId: string;
  initialOutfit: OutfitConfig;
  variant: number;
  onSave: (outfit: OutfitConfig) => void;
  onClose: () => void;
  wardrobePresets?: OutfitConfig[];
  onSaveWardrobe?: (presets: OutfitConfig[]) => void;
}
```

State:
- `currentOutfit: OutfitConfig` — initialized from `initialOutfit`, updated as user makes changes
- `selectedCategory: string` — 'hair' | 'tops' | 'bottoms' | 'shoes' | 'accessories' (default: 'hair')
- `gender: 'M' | 'F'` — from `initialOutfit.gender`, togglable via a button
- `wardrobeSlots: OutfitConfig[]` — up to 4-6 saved preset slots per agent

Layout (use inline styles, matching LayoutEditorPanel.tsx pattern — no CSS files):
```
+-----------------------------------------------+
|  Avatar Builder                          [X]   |
+-----------------------------------------------+
|            |                                   |
|  [Preview  |  [M] [F]    Gender Toggle         |
|   Canvas]  |                                   |
|            |  [Hair] [Tops] [Bottoms] [Shoes]  |
|            |  [Accessories]    Category Tabs    |
|            |                                   |
|            |  +---+---+---+---+                |
|            |  | A | B | C | D |  Icon Grid     |
|            |  +---+---+---+---+  (items)       |
|            |  | E | F | G | H |                |
|            |  +---+---+---+---+                |
|            |                                   |
|            |  Color: [palette swatches grid]   |
|            |                                   |
+-----------------------------------------------+
|  Wardrobe: [1] [2] [3] [+Save]                |
|                              [Save] [Cancel]   |
+-----------------------------------------------+
```

Modal dimensions: ~500px wide, ~450px tall, centered on screen. Dark background (rgba(0,0,0,0.85)), white/light text, matching the LayoutEditorPanel dark UI theme.

**Category tabs:** Buttons at top, active tab highlighted with blue background (#0066cc).

**Gender toggle:** Two buttons [M] [F], active gender highlighted. Changing gender re-filters the catalog and resets clothing to first available item per category if current selection is gender-incompatible.

**Icon grid:** For the selected category, show all CatalogItem entries matching current gender (include 'U' items). Display as small buttons in a grid (4 columns, ~40x40px each). Each button shows the item's displayName as a tooltip. The currently-selected item has a highlighted border. Clicking an item updates `currentOutfit.parts` for the relevant slot.

**Color palette:** Below the icon grid, show the appropriate palette swatches based on selected category:
- Hair category: show HAIR_PALETTE
- Tops/bottoms/shoes/accessories: show CLOTHING_PALETTE
- Always show SKIN_PALETTE in a separate "Skin" section at the top

Each swatch is a small colored square (~20x20px). The currently-selected color has a white border. Clicking a swatch updates `currentOutfit.colors` for the relevant field.

**Preview canvas:** A `<canvas>` element (120x180px) in the left panel. Use a `useEffect` that calls `renderAvatarPreview(ctx, currentOutfit, spriteCache, variant)` whenever `currentOutfit` changes. Get `spriteCache` from `(window as any).spriteCache`.

**Asset loading:** When the user selects a catalog item whose asset is not yet loaded in the sprite cache, load it on demand:
```typescript
const spriteCache = (window as any).spriteCache as SpriteCache;
const requiredAssets = getRequiredAssets(currentOutfit);
for (const assetName of requiredAssets) {
  if (!spriteCache.hasNitroAsset(assetName)) {
    const { nitroFiguresBase } = (window as any).ASSET_URIS;
    await spriteCache.loadNitroAsset(assetName, `${nitroFiguresBase}/${assetName}.png`, `${nitroFiguresBase}/${assetName}.json`);
  }
}
```
Show a brief loading indicator (text "Loading...") while assets load.

**Wardrobe slots:** At the bottom, show 4-6 small buttons representing saved outfit presets. Clicking a slot loads that outfit. A [+Save] button saves the current outfit to the next empty slot (or overwrites the last one if full).

**Save button:** Calls `onSave(currentOutfit)` and closes the modal.
**Cancel button:** Calls `onClose()` without saving.

**Event propagation:** The modal overlay div should have `onClick={(e) => e.stopPropagation()}` to prevent clicks from reaching the canvas behind it.

Add the save handler:
```typescript
const handleBuilderSave = (outfit: OutfitConfig) => {
  if (!builderAvatarId) return;
  // Update avatar spec in place (rAF loop picks up changes)
  avatarManagerRef.current.setAvatarOutfit(builderAvatarId, outfit);
  // Persist via extension host
  const vscodeApi = (window as any).vscodeApi;
  if (vscodeApi) {
    vscodeApi.postMessage({ type: 'saveAvatar', agentId: builderAvatarId, outfit });
  }
  setIsBuilderOpen(false);
  setBuilderAvatarId(null);
};
```

Add the close handler:
```typescript
const handleBuilderClose = () => {
  setIsBuilderOpen(false);
  setBuilderAvatarId(null);
};
```

return (

      return (
        <AvatarBuilderModal
          avatarId={builderAvatarId}
          initialOutfit={avatar.outfit || getDefaultPreset(avatar.variant)}
