Create `src/avatarOutfitConfig.ts` with:

1. **Re-export PartType** from a shared location (or redefine for now):
```typescript
export type PartType = "hrb" | "bd" | "lh" | "lg" | "sh" | "ch" | "ls" | "rh" | "rs" | "hd" | "hr";
```

2. **OutfitConfig interface** — per-avatar outfit definition:
```typescript
export interface OutfitConfig {
  gender: 'M' | 'F';
  parts: {
    hair: { asset: string; setId: number };
    shirt: { asset: string; setId: number };
    pants: { asset: string; setId: number };
    shoes: { asset: string; setId: number };
  };
  colors: {
    skin: string;   // hex
    hair: string;
    shirt: string;
    pants: string;
    shoes: string;
  };
}
```
Note: body/head parts always use `hh_human_body` setId 1, so they are NOT included in OutfitConfig — they are implicit. Head shape is derived from variant index as before.

3. **CatalogItem interface** for the clothing catalog:
```typescript
export interface CatalogItem {
  id: string;
  asset: string;        // cortex-assets figure name
  setId: number;         // Habbo setId for frame key construction
  partType: PartType;    // which body slot (hr, ch, lg, sh)
  category: 'hair' | 'tops' | 'bottoms' | 'shoes' | 'accessories';
  gender: 'M' | 'F' | 'U';
  displayName: string;
}
```

4. **FIGURE_CATALOG** — curated array of ~25-30 CatalogItem entries. Include:
   - Hair: Hair_M_yo (M), Hair_U_Messy setId 2071 (U), Hair_U_Messy setId 2072 (U), Hair_F_Bob (F), Hair_U_Multi_Colour (U) — at least 5 styles
   - Tops: Shirt_M_Tshirt_Plain (M), Shirt_F_Tshirt_Plain (F), Shirt_F_Tshirt_Sleeved (F), Shirt_M_Tshirt_Sleeved (M), Shirt_F_Cardigan (F), Shirt_M_Cardigan (M), Shirt_F_Punk_Shirt (F) — at least 7 tops
   - Bottoms: Trousers_U_Skinny_Jeans (U), Trousers_U_Sraight (U), Trousers_U_runway (U), Trousers_F_Leather_skirt (F) — at least 4
   - Shoes: Shoes_U_Slipons (U), Shoes_F_Schoolshoes (F) — at least 2
   - Accessories: Hat_U_sombrero (U), Hat_U_urban (U) — at least 2 (accessories are optional slots)

   For items with multiple setIds in one asset file (like Hair_U_Messy), create separate catalog entries per setId.

   Hair items map to partType 'hr' (hrb is the back layer — same asset, same setId).
   Shirt items map to partType 'ch' (ls, rs are sleeve layers — same asset, same setId).
   Trouser items map to partType 'lg'.
   Shoe items map to partType 'sh'.

5. **Color palettes** — constant arrays:
```typescript
export const SKIN_PALETTE = ['#FCEBD6', '#F5D6C3', '#EFCFB1', '#D4A574', '#C49060', '#9E6B4A', '#7D5238', '#5C3A1E'];
export const HAIR_PALETTE = ['#1A1A1A', '#4A3728', '#8B6914', '#C4651A', '#D4A017', '#E0E0E0', '#D55B5B', '#5B9BD5', '#5BD55B', '#9B5BD5', '#FF69B4', '#FF8C00'];
export const CLOTHING_PALETTE = ['#FFFFFF', '#CCCCCC', '#666666', '#333333', '#D55B5B', '#FF8C00', '#D5D55B', '#5BD55B', '#5B9BD5', '#3B5998', '#9B5BD5', '#4B0082', '#8B4513', '#556B2F', '#800020', '#2C2C2C'];
```

6. **DEFAULT_PRESETS** — array of 8 OutfitConfig objects providing visual variety. Use the existing 6 VARIANT_OUTFITS color combinations as a starting point, plus 2 more with female gender and different clothing. These presets will be assigned to new agents based on `variant % DEFAULT_PRESETS.length`.

7. **Helper functions:**
   - `getCatalogByCategory(gender: 'M' | 'F'): Map<string, CatalogItem[]>` — filter by gender (include 'U' items for both), group by category
   - `getCatalogForSlot(category: string, gender: 'M' | 'F'): CatalogItem[]` — convenience filter
   - `getDefaultPreset(variant: number): OutfitConfig` — returns `DEFAULT_PRESETS[variant % DEFAULT_PRESETS.length]`
   - `outfitToFigureParts(outfit: OutfitConfig): Record<PartType, { asset: string; setId: number }>` — converts an OutfitConfig into the FIGURE_PARTS format the renderer expects. Body parts always map to hh_human_body setId 1. Hair maps hr AND hrb. Shirt maps ch, ls, AND rs. Pants maps lg. Shoes maps sh.
   - `getRequiredAssets(outfit: OutfitConfig): string[]` — returns list of unique asset names needed (for lazy loading)
