Add editor state types and functions to isoLayoutEditor.ts:

**EditorMode type:**
```typescript
export type EditorMode = 'view' | 'paint' | 'color';
```

**EditorState interface:**
```typescript
export interface EditorState {
  mode: EditorMode;
  hoveredTile: { x: number; y: number; z: number } | null;
  selectedColor: HsbColor;
}
```

**toggleTileWalkability(grid: TileGrid, tileX: number, tileY: number): void**
- Check if grid.tiles[tileY][tileX] exists
- If tile exists (walkable), set to null (make void)
- If null (void), create new tile {height: 0} (make walkable at ground level)
- Mutate grid in place (matches existing React ref patterns)

**setTileColor(tileColorMap: Map&lt;string, HsbColor&gt;, tileX: number, tileY: number, color: HsbColor): void**
- Create key: `${tileX},${tileY}`
- Set tileColorMap.set(key, color)
- Mutate map in place

**gridToHeightmap(grid: TileGrid): string**
- Map each row to string using: row.map(tile => tile ? String(tile.height) : 'x').join('')
- Join rows with '\n'
- Return heightmap string for save/persistence

Import HsbColor from isoTypes.ts. These are pure mutation functions (no React dependencies).

Add editor mode support to RoomCanvas component:

**Add editorMode prop to RoomCanvasProps:**
```typescript
interface RoomCanvasProps {
  heightmap: string;
  editorMode?: EditorMode; // Optional, defaults to 'view'
}
```

**Add editor state to renderState ref:**
```typescript
renderState = useRef({
  // ... existing fields ...
  editorState: {
    mode: 'view' as EditorMode,
    hoveredTile: null as { x: number; y: number; z: number } | null,
    selectedColor: { h: 200, s: 50, b: 50 } as HsbColor,
  },
  grid: null as TileGrid | null,
  tileColorMap: new Map<string, HsbColor>(),
});
```

**Add mouse event handlers to canvas element:**
- onMouseMove: Call getHoveredTile(), update renderState.current.editorState.hoveredTile
- onClick: If editorMode === 'paint', call toggleTileWalkability(); if 'color', call setTileColor()
- onMouseLeave: Clear hoveredTile (set to null)

**Store grid in renderState:**
After parseHeightmap() call in useEffect, store result: `renderState.current.grid = grid;`

**Re-render offscreenCanvas on edit:**
After toggleTileWalkability or setTileColor, call preRenderRoom() again with updated grid/tileColorMap

**Draw hover highlight in frame() loop:**
After blitting offscreen canvas, before avatars, check if editorState.hoveredTile exists. If yes, call drawHoverHighlight() with hovered tile coordinates.

Pattern: Editor state lives in useRef (mutable, not useState) to avoid triggering React re-renders on every mouse move. Only offscreen canvas regeneration triggers on actual edits (paint/color changes).

Important: Pass tileColorMap to preRenderRoom() so per-tile colors are respected. This parameter already exists in preRenderRoom signature but is currently unused in RoomCanvas.tsx.
