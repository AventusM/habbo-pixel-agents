  <name>Task 1: Implement mouse-to-tile conversion module</name>
  <files>src/isoLayoutEditor.ts</files>
  <action>
Create isoLayoutEditor.ts with two core functions:

**getHoveredTile(event: React.MouseEvent&lt;HTMLCanvasElement&gt;, cameraOrigin: {x, y}): {tileX, tileY} | null**
- Use getBoundingClientRect() to get canvas position
- Calculate scale factors: scaleX = canvas.width / rect.width, scaleY = canvas.height / rect.height
- Get mouse position: mouseX = (event.clientX - rect.left) * scaleX
- Subtract camera origin: adjX = mouseX - cameraOrigin.x, adjY = mouseY - cameraOrigin.y
- Call screenToTile(adjX, adjY) from isometricMath.ts
- Floor result to integer tile indices
- Return null if outside grid bounds, otherwise return {tileX: Math.floor(x), tileY: Math.floor(y)}

**drawHoverHighlight(ctx: CanvasRenderingContext2D, tileX: number, tileY: number, tileZ: number, cameraOrigin: {x, y}): void**
- Call tileToScreen(tileX, tileY, tileZ) to get screen position
- Save context state
- Translate by cameraOrigin.x and cameraOrigin.y
- Set strokeStyle = 'rgba(255, 255, 100, 0.8)' (yellow, 60% opacity per research)
- Set lineWidth = 2
- Draw rhombus outline using same path as tile rendering (beginPath, moveTo top vertex, lineTo right/bottom/left vertices, closePath, stroke)
- Restore context state

Import TILE_W_HALF, TILE_H_HALF from isometricMath.ts. No React dependencies (pure Canvas 2D logic).

Pattern note: This is the SAME rhombus shape as tiles from Phase 2, just stroke instead of fill. Copy the rhombus path logic from isoTileRenderer.ts if helpful.
