  <name>Task 1: Implement walk-frame offset delta correction for non-walk parts</name>
  <files>src/isoAvatarRenderer.ts</files>
  <action>
Add a `getBodyWalkDelta` function that computes the offset difference between the current walk-frame body sprite and the standing body sprite. This delta represents how much the body "bounces" during each walk frame.

Implementation:
1. Add function `getBodyWalkDelta(spriteCache, direction, frame, figureParts)` that:
   - Gets the mapped direction via `mapBodyDirection(direction).dir`
   - Looks up `h_std_bd_{setId}_{dir}_0` offset (standing reference)
   - Looks up `h_wlk_bd_{setId}_{dir}_{frame}` offset (current walk frame)
   - Returns `{ dx: wlk.offsetX - std.offsetX, dy: wlk.offsetY - std.offsetY }`
   - Returns `{ dx: 0, dy: 0 }` if either frame is missing (safe fallback)
   - Uses `figureParts['bd'].setId` for the setId

2. In `createNitroAvatarRenderable`, inside the render loop for non-face parts (after line 684 where `frame` is resolved), add:
   - After obtaining the final `frame` for a part, check if `stateForFrame === 'walk' && !WALK_PARTS.has(part)`
   - If so, call `getBodyWalkDelta(spriteCache, spec.direction, spec.frame, figureParts)`
   - Create a modified frame object: `frame = { ...frame, offsetX: frame.offsetX + delta.dx, offsetY: frame.offsetY + delta.dy }`
   - This makes ch (chest), hd (head), hr (hair), and hrb (hair back) track the body bounce

3. For face parts (ey, fc) inside the face rendering block (around line 659), apply the same delta correction:
   - When `stateForFrame === 'walk'`, compute delta and adjust faceFrame offset before calling drawTintedBodyPart
   - Create modified faceFrame: `const adjustedFrame = { ...faceFrame, offsetX: faceFrame.offsetX + delta.dx, offsetY: faceFrame.offsetY + delta.dy }`

IMPORTANT: Do NOT apply delta when state is idle or sit (delta will be 0,0 from fallback, but skip the computation entirely for clarity). Guard with `if (stateForFrame === 'walk')`.

IMPORTANT: The delta is computed in pre-flip space. The existing flip logic in drawTintedBodyPart handles coordinate transformation, so do NOT negate deltaX for flipped directions.

Update the comment on line 292 to reflect the fix: "ch, hd, hr, hrb use std frames but track body walk-frame delta during walk".
